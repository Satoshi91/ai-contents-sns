## 2. 画面詳細設計

### 2.1 ランディングページ (/)

#### 概要
未認証ユーザーが最初に訪れるページ

#### レイアウト
```
┌─────────────────────────────────────┐
│         ヘッダー                     │
│  [ロゴ]              [ログイン] [登録]│
├─────────────────────────────────────┤
│                                     │
│      [アプリタイトル]                │
│   「今起きていることを共有しよう」     │
│                                     │
│   [サインアップボタン]               │
│   [ログインボタン]                  │
│                                     │
└─────────────────────────────────────┘
```

#### 機能
- サインアップ画面への遷移
- ログイン画面への遷移

```
│  パスワード（確認）                  │
│  [___________________________]     │
│                                     │
│  ユーザー名                         │
│  @[__________________________]     │
│  ※3-15文字、英数字とアンダースコア   │
│                                     │
│  表示名                             │

│                                     │
│      [アカウント作成]                │

```

#### バリデーション
- メールアドレス: 有効な形式
- パスワード: 8文字以上
- ユーザー名: 3-15文字、英数字とアンダースコアのみ、重複チェック
- 表示名: 1-50文字

## 3. UIコンポーネント設計

### 3.1 共通コンポーネント

#### Header
- ロゴ
- ナビゲーションメニュー
- ユーザーアバター

#### PostCard
- ユーザー情報
- 投稿内容
- タイムスタンプ
- アクションボタン

#### Button
- Primary: 主要アクション
- Secondary: 副次的アクション
- Danger: 削除などの危険なアクション

#### Input
- テキスト入力
- パスワード入力
- テキストエリア

#### Modal
- オーバーレイ
- コンテンツエリア
- クローズボタン

### 3.2 レスポンシブデザイン

#### ブレークポイント
- Mobile: < 640px
- Tablet: 640px - 1024px
- Desktop: > 1024px

#### モバイル対応
- ハンバーガーメニュー
- タッチ最適化
- スワイプジェスチャー

## 4. 色とテーマ

### 4.1 カラーパレット

#### ライトモード
- Primary: #1DA1F2 (Twitter Blue)
- Secondary: #14171A
- Background: #FFFFFF
- Surface: #F7F9FA
- Text: #14171A
- TextSecondary: #536471
- Border: #EFF3F4
- Error: #F4212E
- Success: #00BA7C

#### ダークモード
- Primary: #1DA1F2
- Secondary: #FFFFFF
- Background: #000000
- Surface: #16181C
- Text: #FFFFFF
- TextSecondary: #8B98A5
- Border: #2F3336
- Error: #F4212E
- Success: #00BA7C

### 4.2 タイポグラフィ

- 見出し1: 24px, Bold
- 見出し2: 20px, Bold
- 本文: 16px, Regular
- キャプション: 14px, Regular
- 小文字: 12px, Regular

## 5. アニメーション

### 5.1 トランジション
- ページ遷移: フェードイン/アウト (300ms)
- モーダル表示: スライドアップ (200ms)
- ボタンホバー: 背景色変更 (150ms)

### 5.2 ローディング
- スピナー: 投稿送信時
- スケルトンスクリーン: タイムライン読み込み時
- プログレスバー: 画像アップロード時

## 6. アクセシビリティ

### 6.1 基本要件
- WCAG 2.1 Level AA準拠
- キーボードナビゲーション対応
- スクリーンリーダー対応
- 適切なARIAラベル

### 6.2 実装項目
- フォーカス管理
- エラーメッセージの明確化
- 十分なコントラスト比
- 代替テキストの提供

### 7.1 バリデーションエラー

### 7.2 システムエラー
